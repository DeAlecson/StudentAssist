leaderboard.js

app.js

ROUTING STRUCTURE
=================

#/                  → Hub (module selector)
#/ict162            → Module home (ICT162)
#/ict162/learn      → Learn mode selector
#/ict162/learn/su1  → Learn specific study unit
#/ict162/quiz       → Quiz mode selector
#/ict162/quiz/su1   → Quiz specific study unit
#/ict162/assessment → Assessment mode
#/ict162/mixed      → Mixed (exam) mode

Same pattern for sst101 and acc202


STORAGE STRUCTURE
=================

LocalStorage Keys:
- sa_initialized           → Initialization flag
- sa_{moduleId}_progress   → Module progress object
- sa_settings              → Global settings

SessionStorage Keys:
- sa_api_key               → Anthropic API key (cleared on tab close)

Progress Object:
{
  version: '1.0.0',
  xp: 0,
  level: 1,
  streak: 0,
  lastActiveDate: null,
  completedLessons: [],
  completedQuizzes: [],
  completedFlashcards: [],
  masteryMap: {},
  wrongAnswers: [],
  weakTopics: [],
  attempts: {}
}

EVENT BINDINGS
==============

Button Clicks:
- #back-btn         → Navigate to hub
- #nav-toggle-btn   → Open side nav
- #nav-close        → Close side nav
- #settings-btn     → Open settings
- #settings-close   → Close settings
- #lb-btn           → Open leaderboard
- #lb-close         → Close leaderboard
- #api-key-save     → Save API key
- #api-key-clear    → Clear API key
- #display-name-save → Save display name
- #export-btn       → Export data as JSON
- #reset-all-btn    → Reset all data (with confirm)
- #sn-reset-btn     → Reset module progress (with confirm)
- .sn-btn[data-route] → Navigate to mode routes
- .sn-mod-card      → Switch modules
- .lb-filter-btn    → Filter leaderboard by module

Overlays:
- #nav-overlay      → Close side nav when clicked
- #settings-overlay → Close settings when clicked
- #leaderboard-overlay → Close leaderboard when clicked

Hash Changes:
- hashchange event  → Route resolution

Form Inputs:
- All input fields auto-save on button click
